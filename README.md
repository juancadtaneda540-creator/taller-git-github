Objetivo del proyecto
Aprender y aplicar Git y GitHub mediante un flujo profesional usando ramas, commits, PRs, conflictos y documentación.

¿Qué es Git?
Git es un sistema de control de versiones que permite administrar los cambios de un proyecto.
Ayuda a guardar historial, volver atrás, crear ramas, trabajar en equipo y evitar perder información.
Fue creado por Linus Torvalds en 2005.
Con Git puedes trabajar localmente sin internet, y luego sincronizar tus cambios.
Cada cambio queda registrado como un commit.
Permite flujos de trabajo profesionales usados en empresas reales.
Facilita el trabajo colaborativo.
Es una herramienta esencial para desarrolladores.

10 comandos de Git explicados
git init → Inicializa un repositorio.
git status → Muestra los archivos modificados.
git add . → Agrega archivos al área de preparación.
git commit -m "" → Guarda un registro de cambios.
git log → Muestra historial de commits.
git branch → Lista ramas.
git checkout -b nombre → Crea y cambia a una rama.
git push → Sube cambios al remoto.
git pull → Descarga cambios del remoto.
git merge → Une ramas.

Flujo de trabajo recomendado
Crear rama.
Hacer cambios.
Agregar los cambios (git add).
Hacer commit.
Subir la rama (git push -u origin).
Crear un Pull Request.
Pedir revisión y hacer merge.
